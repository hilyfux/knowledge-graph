# Architecture Notes — Source Code Research Findings

Based on deep study of Claude Code source (cc-haha) and caveman project, 2026-04-09.

## Claude Code Internal Mechanisms (verified from source)

### CLAUDE.md Loading
- Priority: later loaded = higher attention. Order: Managed → User → Project (root→CWD) → Local
- Subdirectory CLAUDE.md: lazily loaded via nested traversal when Claude accesses files in that dir
- @include: max depth 5, circular ref prevented, external includes need approval
- Size: 40K chars soft limit (flagged but not truncated)

### Hooks
- 27 hook events available (we use 8)
- UserPromptSubmit: only supports `additionalContext`, NOT prompt modification
- PreCompact: can inject instructions into compaction prompt
- PostCompact: can re-inject context after compaction
- Exit code 2 = blocking error; exit 0 = success
- Permission precedence: deny > ask > allow > passthrough

### Context Survival
- System prompt (CLAUDE.md, rules): auto-rebuilt after compact ✅
- @include content: auto-rebuilt (part of system prompt) ✅
- SessionStart hook output: LOST on compact ❌ (becomes summary)
- Nested memory attachments: cache cleared, re-loaded on next file access ✅
- Session Memory (built-in): survives compact, customizable template

### Compaction
- 9-section summary generated by API call
- Post-compact: up to 5 files restored (50K token budget)
- readFileState + loadedNestedMemoryPaths cleared
- SessionStart hooks re-run with source='compact'

## Unused Hooks (potential future use)
- FileChanged: track file changes from IDE/external editors
- CwdChanged: inject knowledge when user navigates directories
- SessionEnd: cleanup or final analysis
- PreToolUse.updatedInput: modify tool inputs before execution

## Caveman Compression Insights
- Validation-driven: heading count, code blocks, URLs must be exact
- Fix prompt (not recompress): surgical fix of specific errors
- ~45% input token savings on prose, ~25-35% on structured files
- Auto-Clarity escape: know when NOT to compress (security, irreversible actions)

## Session Memory (built-in, potential integration)
- Template customizable at ~/.claude/session-memory/config/template.md
- 9 sections: Title, State, Task, Files, Workflow, Errors, Docs, Learnings, Results
- Could add KG-specific sections for cross-compact state preservation
- Gated behind feature flag, may not be universally available

## Design Decisions Made
1. Keep subdirectory CLAUDE.md (native lazy loading, O(1) lookup) — not centralize to rules/
2. @include for knowledge-index (system prompt level) — not hook injection
3. PreCompact guidance (influence compaction) — not just PostCompact recovery
4. ≤20 line CLAUDE.md with compression rules — not caveman-compress tool (our files already terse)
